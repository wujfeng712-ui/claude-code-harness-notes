# s01 · Agent Loop — the minimal runnable loop

> **Motto: one loop + Bash is an agent.**

## What this lesson solves

A model only "emits commands" and then stops, waiting for someone to run them and paste the result back. s01 automates that middle layer: a `while stop_reason == "tool_use"` decide → execute → feed-back loop lets the model keep reasoning until it decides to stop on its own. This loop is the fixed point of the whole 20-lesson series — every later mechanism layers around it without rewriting it.

## How the mechanism works

The core is a single loop: model decides → tools execute → results feed back, until the model says "I'm done."

```python
while stop_reason == "tool_use":
    response = call_model(messages)        # 1. model decides
    if response.stop_reason != "tool_use":
        break                              # model says "I'm done" -> exit
    for tool_call in response.tool_calls:  # 2. execute tools
        result = TOOL_HANDLERS[tool_call.name](**tool_call.input)
        messages.append(tool_result(result))  # 3. results feed back
```

![Core loop: model decides -> tools execute -> results feed back](../../../images/diagrams/01-core-loop.svg)

- `messages` keeps accumulating the conversation history; `TOOL_HANDLERS` provides the executable tools.
- Stopping is decided by `stop_reason`: keep going while it's `tool_use`, otherwise exit.

## Key insights

- This loop stays **byte-for-byte unchanged from lesson 1 through lesson 20** — the single most important design claim of the project.
- Automating the "human copy-paste" upgrades the model from "emitting commands" to "reasoning repeatedly until it decides to stop."
