# s06 · Subagent (Context Isolation)

> **Motto: Break big tasks into small ones, each with a clean context.**

## What this lesson solves

Once the main agent piles all its exploration, trial-and-error, and intermediate artifacts into one `messages[]`, the context quickly gets polluted with noise and overflows. A subagent hands a big task off to a brand-new, clean conversation and only brings back the final conclusion, so the main thread never gets dragged down by the middle steps.

## How the mechanism works

The `task` tool spawns a subagent with a **brand-new `messages[]`**, lets it run its own loop, and on completion **returns only the final summary**, discarding everything in between. Like "open a new terminal, finish the job, close it, carry the result back to the main terminal."

```
main agent messages[] ─── task("research X") ──▶ subagent fresh messages[]
                                                   │ its own loop: explore/try/produce
                                                   ▼
main agent ◀──────── only final summary ───────── everything in between discarded
```

⚠️ Key distinction: what's isolated is the **context**, not the **permissions** — the subagent's tool calls still pass through the `PreToolUse` hook. The subagent has no `task` tool, to prevent infinite recursion.

## Key insights

- It isolates context, not permissions: the subagent is still bound by the `PreToolUse` hook.
- The subagent holds no `task` tool, structurally ruling out infinite recursion.
- "Return only the summary, discard the process" trades a throwaway dirty context for a clean main thread.
