# s07 · On-Demand Skill Loading

> **Motto: Load it when you need it, don't stuff it all into the prompt.**

## What this lesson solves

Large documents (React conventions, SQL style, API design…) waste precious context if you stuff them all into the system prompt, and most turns never use them. On-demand loading lets the agent always "know what capabilities exist" while only injecting the full content when it's actually used.

## How the mechanism works

Two-layer loading: at startup only the cheap "catalog" stays resident; at runtime the expensive "body" is pulled on demand.

```
startup: scan skills/, keep  name + description (~100 tokens)  resident in SYSTEM   ← cheap catalog
runtime: load_skill(name) injects the  full SKILL.md (~2000 tokens)                 ← expensive content on demand
```

This keeps the agent always aware of what it can do (low cost), with full content loaded only when used (on-demand cost).

## Key insights

- "Knowing it exists" and "loading it into context" are two different things — the catalog stays resident cheaply, the body loads expensively on demand.
- The description is the trigger: write it precisely, and the agent calls `load_skill` at the right moment.
