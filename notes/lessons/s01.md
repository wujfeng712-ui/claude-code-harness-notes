# s01 · Agent Loop — 最小可运行的循环

> **Motto：一个循环 + Bash 就是一个 Agent。**

## 这一课解决什么问题

Model 只会「输出命令」，停在那里等人去执行、再把结果粘回去。s01 把这个中间层自动化掉：用一个 `while stop_reason == "tool_use"` 的决策→执行→回填循环，让模型能反复推理直到自己决定停止。这个循环是整个 20 课系列的不动点，后面所有机制都叠加在它周围而不改写它。

## 机制怎么实现

核心就是一个循环：模型决策 → 执行工具 → 结果回填，直到模型说「我说完了」。

```python
while stop_reason == "tool_use":
    response = call_model(messages)        # ① 模型决策
    if response.stop_reason != "tool_use":
        break                              # 模型说「我说完了」→ 退出
    for tool_call in response.tool_calls:  # ② 执行工具
        result = TOOL_HANDLERS[tool_call.name](**tool_call.input)
        messages.append(tool_result(result))  # ③ 结果回填
```

![核心循环：模型决策 → 执行工具 → 结果回填](../../images/diagrams/01-core-loop.svg)

- `messages` 持续累积对话历史，`TOOL_HANDLERS` 提供可执行的工具。
- 停止判定靠 `stop_reason`：只要还是 `tool_use` 就继续，否则退出。

## 关键洞见

- 这段循环从第 1 课到第 20 课**一字未改**——这是全项目最重要的设计主张。
- 把「人工复制粘贴」自动化，模型就从「输出命令」升级成「反复推理直到决策停止」。
