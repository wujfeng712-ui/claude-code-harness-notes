# s02 · 工具分发 — 从 1 个工具到 N 个工具

> **Motto：加工具只加 handler，循环不动。**

## 这一课解决什么问题

s01 的循环只会跑一个硬编码的工具。要支持多个工具，又不想每加一个就改循环。s02 把工具执行那一行从硬编码改成查表分发：新增工具 = 加一条 schema + 一个 handler，循环本身完全不用动。

## 机制怎么实现

唯一的改动在工具执行那一行：从硬编码 `run_bash(...)` 改为查表分发。

```python
result = TOOL_HANDLERS[block.name](**block.input)
```

- `TOOL_HANDLERS` 是一张「工具名 → handler 函数」的分发表（dispatch map）。
- 新增工具只需注册到表里，循环逻辑保持不变。
- `safe_path()` 校验文件类工具的路径不能逃出工作区，防止目录逃逸。

## 关键洞见

- 把「执行哪个工具」的判断从循环里抽出来变成查表，循环就对工具数量解耦了。
- 文件类工具必须做路径校验——工具能力越强，越要在代码层兜底。
