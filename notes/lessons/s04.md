# s04 · Hooks — 转折点

> **Motto：挂在循环上，不写进循环里。**

## 这一课解决什么问题

s03 把权限检查硬编码进了循环。如果每加一个机制（权限、日志、压缩、记忆…）都往循环里塞 `if-else`，循环很快变成谁都不敢碰的怪物。s04 引入 hooks（钩子）：把检查搬出循环，循环成为稳定内核，任何扩展都靠「挂钩」接入。**这一课是整个项目的工程支点。**

## 机制怎么实现

把 s03 硬编码在循环里的 `check_permission()` 搬进 `permission_hook`，循环改为：

```python
blocked = trigger_hooks("PreToolUse", block)
```

`HOOKS` 是事件注册表，四个事件让任何扩展都靠挂钩接入：

- `UserPromptSubmit` — 用户输入后、模型调用前（注入上下文）
- `PreToolUse` — 工具执行前（权限、日志）
- `PostToolUse` — 工具执行后（输出检查、副作用）
- `Stop` — 循环退出前（收尾、可强制续跑）

## 关键洞见

- 引入钩子后，循环从「会不断膨胀的逻辑」变成「稳定内核 + 扩展点」。
- 这是后面所有机制（s05 起）能干净叠加的前提——它们全挂在这四个事件上。
