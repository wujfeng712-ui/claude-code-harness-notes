# s05 · TodoWrite — 规划工具

> **Motto：没有计划的 agent 走哪算哪。**

## 这一课解决什么问题

模型容易被即时的工具反馈带偏，做着做着忘了最初要干嘛。s05 加入 `todo_write`：它不执行任何实际操作，增加的不是执行能力而是**规划能力**——强制模型动手前先把步骤过一遍，并在跑偏时把它拉回初衷。

## 机制怎么实现

- `todo_write` 把计划写进内存里的 `CURRENT_TODOS`，本身不产生任何副作用。
- 配合 nag reminder（督促提醒）：用 `rounds_since_todo` 计数，当 `>= 3`（三轮没更新 TODO）时自动注入一条提醒，防止模型忘记初衷。

```
执行工具 → rounds_since_todo += 1
            │
            ▼
     >= 3 ？ ──是──► 注入「该更新一下 TODO 了」提醒
            │
            否 → 继续
```

## 关键洞见

- 规划工具的价值不在「做了什么」，而在「逼模型先想清楚再做」。
- 即时反馈会让模型短视；三轮督促是一个便宜的纠偏机制。
