# s07 · Skill 按需加载

> **Motto：用到时再加载，别全塞 prompt 里。**

## 这一课解决什么问题

大文档（React 规范、SQL 风格、API 设计…）如果全塞进 system prompt，会白白吃掉宝贵的上下文，而且大多数轮次根本用不到。按需加载让 agent 始终「知道有什么能力」，但完整内容只在真正用到时才注入。

## 机制怎么实现

两层加载：启动时只常驻廉价的「目录」，运行时才按需拉取昂贵的「正文」。

```
启动时：扫描 skills/，把  name + description（~100 token）  常驻 SYSTEM   ← 廉价目录
运行时：load_skill(name)，才注入  完整 SKILL.md（~2000 token）          ← 昂贵内容按需
```

让 agent 始终「知道有什么能力」（低成本），完整内容只在用到时加载（按需成本）。

## 关键洞见

- 「知道存在」和「装进上下文」是两件事——目录廉价常驻，正文昂贵按需。
- description 是触发器：写得准，agent 才会在对的时机调 `load_skill`。
