# s10 · System Prompt 运行时组装

> **Motto：Prompt 是组装出来的，不是写死的。**

## 这一课解决什么问题

把 system prompt 写成一个硬编码字符串，会让它无法随真实环境变化——记忆库存不存在、有没有 skill，全靠不上。这一课把 SYSTEM prompt 升级为**按真实状态组装的配置**。

## 机制怎么实现

SYSTEM prompt 从硬编码字符串升级为**按真实状态组装的配置**。`PROMPT_SECTIONS` 按需拼接（如 `.memory/MEMORY.md` 存在才加载 memory 段），用 `json.dumps`（而非 `hash()`）做 cache key 避免进程内重复拼接。是否加载某段由**真实文件/工具状态**决定，而非消息里的关键词猜测。

```
真实状态 ── 文件存在? 工具可用? ──▶ assemble_system_prompt
PROMPT_SECTIONS ─ 按需拼接 ──────────▶  组装结果
                                         │ json.dumps 做 cache key
                                         ▼  缓存命中则复用，不重复拼接
```

## 关键洞见

- 加载哪一段由**真实文件/工具状态**决定，不靠在消息里猜关键词。
- cache key 用 `json.dumps` 而非 `hash()`，保证同一状态稳定命中、不重复拼接。
