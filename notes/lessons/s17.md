# s17 · 自治 Agent — 自己看板，自己认领

> **Motto：自己看板，自己认领。**

## 这一课解决什么问题

s15/s16 的队友是被动的：等 Lead 把任务塞过来才动。一旦 Lead 成了唯一的调度瓶颈，并行度就被锁死。s17 让队友**自己看板、自己认领**——进入一个 `WORK → IDLE → SHUTDOWN` 的生命周期，空闲时主动去任务图里捞活。

## 机制怎么实现

队友进入生命周期循环：

```
WORK: inbox → LLM → tools →（还有 tool_use? 继续）→（做完? → IDLE）
IDLE: 每5秒轮询 → inbox有消息?     → 回WORK
                → 有未认领任务?     → claim → 回WORK
                → 60秒没活?        → SHUTDOWN
```

- IDLE 阶段 `scan_unclaimed_tasks()` 找 `pending + 无 owner + can_start`（依赖已满足）的任务，自动认领。
- 压缩后会自动重注入 `<identity>You are {name}…`，防止队友在上下文被裁剪后「忘了自己是谁」。

## 关键洞见

- 把调度从「Lead 推」翻转成「队友拉」，Lead 不再是并行度的瓶颈。
- 自治需要明确的退出条件：60 秒无活就 SHUTDOWN，避免空转烧钱。
- 身份要在压缩后重注入——自治 Agent 必须始终知道自己是谁、该认领什么。
