# s18 · Worktree 隔离 — 各干各的，互不干扰

> **Motto：各干各的，互不干扰。**

## 这一课解决什么问题

多个队友并行开发，如果都在同一个工作目录里改文件，彼此的改动会互相覆盖、冲突。s18 用 `git worktree` 给每个任务一个**独立目录 + 独立分支**，让并行开发在文件层面真正互不干扰。

## 机制怎么实现

- `git worktree add` 给每个任务建独立目录 `.worktrees/{name}/` + 分支 `wt/{name}`，任务用 `worktree` 字段绑定。
- 队友的 bash/read/write 自动在该 cwd 下执行，**并行开发文件互不冲突**。
- `validate_worktree_name()` 只允许 `[A-Za-z0-9._-]`，防路径穿越。
- 删除前检查未提交改动，有改动则拒删（除非显式 `discard_changes`）。

```
task A (worktree=feat-a) ─► .worktrees/feat-a/  分支 wt/feat-a
task B (worktree=feat-b) ─► .worktrees/feat-b/  分支 wt/feat-b
                              │
        队友 A 的 bash/read/write 自动落在 feat-a/ ── 互不污染
```

## 关键洞见

- 任务 ↔ 目录绑定，把「并行协作」从逻辑层落到文件系统层，冲突在物理上被隔开。
- 名字校验 + 删除前脏检查，是隔离机制的安全边界：防穿越、防误删未提交的工作。
- worktree 让每个队友天然拥有自己的分支，合并由上层决定，开发期互不阻塞。
